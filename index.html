THE DIM HORSEMAN

General-purpose logics

  • Near-Threshold Voltage Processors

Performance cost by dropping operating voltage can be offset by incorporating more processors. Not robust to fabrication variation.

  • Bigger Caches

Use dark silicon area for caches. Increased cache sizes can carry both performance and energy benefits for miss-intensive applications.

  • CGRA

A recurring dim alternative is the use of reconfigurable logic.

The idea is to naturally lay out the datapaths of the computation in space to avoid the multiplexing costs that are inherent to processor pipelines.

The duty cycle of CGRA elements is very low, making it a potential fit exploiting dark silicon.

Like to look at old designs with fresher eyes.

GPU Overview

GPU Overview

Bunch of threads: “warp” in Nvidia, “wavefront” in AMD

  • “warp” is the smallest executable unit of code, processes a single instruction over all of the threads in it at the same time, is the minimum size of the data processed in SIMD fashion
  • sp: streaming processor

CUDA

  • GPU hardware

    • SP: streaming processor
    • SM streaming multiprocessor = SP+warp scheduler+register+shared memory

  • CUDA software:

    • thread)block)grid
    • warp: thread SIMT
      • SIMD要求所有的vector element在一个统一的同步组里同步的执行，而SIMT允许线程们在一个warp中独立的执行 ，SIMT更灵活。SIMT中每个thread拥有自己的instruction address counter、状态寄存器和独立的执行路径
  • 同一个block中的threads必然在同一个SM中SIMT并行.block一旦被分配好SM，该block就会一直驻留在该SM中，直到执行结束。一个SM可以同时拥有多个blocks，但需要seriel执行。
  • 一个SP可以执行一个thread，一个warp需要占用一个SM运行，多个warps需要轮流进入SM。active warps被SM资源限制。当一个warp空闲时，SM就可以调度驻留在该SM中另一个可用warp。
  • SM的硬件warp scheduler负责调度。

Nvidia

Tesla/Fermi/Kepler/Maxwell/Pascal

Tesla:

  • released in 2006
  • Nvidia’s first microarchitecture to implement unified shaders 着色器
  • replaced the old fixed-pipeline) microarchitectures

Fermi:

  • release to retail on April 2010.

  • An SM consists of 32 CUDA cores.

Kepler: April 2012

  • An SM consists of 192 CUDA cores.
  • Nvidia’s first microarchitecture to focus on energy efficiency.

Maxwell: February 18, 2014

  • An SM consists of 128 CUDA cores.

Pascal: April 2016

The Pascal architecture features High Bandwidth Memory, Unified Memory, and NVLink.

  • An SM consists of 128 CUDA cores.

NUMA (non-uniform memory access): a computer memory design in multiprocessing, where the memory access time depends on the memory location relative to the processor. Under NUMA, a processor can access its own local memory faster than non-local memory, that is, memory local to another processor or memory shared between processors. In a NUMA system, processors, memory, and I/O are grouped together into cells. The latency and bandwidth characteristics of communication within a cell are “fast” while going outside a cell is “slow”. Since the memory in NUMA systems is physically distributed but logically shared, these systems offer better performance to applications that are optimized to use their features. For non-optimized applications, they still offer better performance since the default behavior is designed to be benign if not beneficial and they still have access to much larger shared resources of memory, CPUs, and disk space.

WHAT is the difference between a scratchpad and a cache?

  • Scratchpad memory: SRAM. Next closest to ALU after the processor registers, similar to L1 cache. A system with scratchpads is a system with NUMA latencies, because the memory access latencies to the different scratchpads and the main memory vary. A scratchpad commonly does not contain a copy of data that is also stored in the main memory. “manually” managed, explicitly addresses, writing and retrieving results. Deterministic.
  • Cache: SRAM. hardware-managed. “automatically” managed in hardware. (But if misses from main memory and fetches from disk, OS gets involved.)
  • challenges
    • no explicit interfaces
      • hard for software: techs like thread scheduling, cache bypassing/prioritization, prefetching
      • inter-CTA (or thread blocks), inter-warp, intra-warp locality
      • not portable: fine-grained, across different architecture with a different CTA scheduler
    • application’s access semantics miss from software-transparent arch: NUMA locality (place data close to threads using it), CTA scheduling (scheduling CTAs share data at the same core)
